# 小米 MIMO Token 额度申请说明

## 04 项目描述

我构建了一个面向普通消费者的多模态维权证据链 Agent：MIMO ClaimPilot Agent。它把聊天截图、订单截图、物流记录、付款凭证、合同文本等分散材料，整理为时间线、关键事实、责任点、证据清单、申诉策略和投诉文书。

核心痛点是：普通人并不缺证据，而是缺少把证据组织成平台客服、商家、调解方能快速理解的结构化材料的能力。该 Agent 通过证据抽取、时间线归纳、责任分析、策略生成、文书生成等多个步骤协作，把非结构化材料转成可提交的维权包。

项目当前已提供 FastAPI 后端、网页 Demo、离线规则引擎、测试、CI，以及 MIMO/OpenAI-compatible API 接入位。获得 token 后，将用于真实多模态材料理解、长上下文证据归纳、多版本申诉文书生成、不同平台规则适配和批量案例评估。

## 05 使用证明与影响力证明

- 已落地为可运行 GitHub 项目，包含后端 API、前端 Demo、示例输入、测试和 CI。
- 默认离线模式可以无 Key 演示完整工作流，方便评审直接体验。
- 配置 MIMO API Key 后可切换为真实模型推理。
- 目标场景高频且具体：售后争议、快递延误、课程退款、租房纠纷、二手交易纠纷。
- token 消耗来自真实 Agent 工作流：材料理解、证据结构化、策略生成、文书生成、结果复核。
